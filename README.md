# white-ip

Демон для роутеров Keenetic (Entware) на Go, который следит за WAN-интерфейсом и
добивается «белого» (не-NAT) IP от провайдера.

Периодически сравнивается IP-адрес, полученный WAN-интерфейсом от провайдера, с
адресом, который видно снаружи (через сторонний сервис проверки IP). Если адреса
не совпадают — значит провайдер выдал серый IP за своим NAT. В этом случае
программа на 3 секунды выключает WAN-интерфейс через RCI, включает его снова,
дожидается восстановления соединения и таймера, затем повторяет проверку — пока
не будет получен белый IP или не закончатся попытки в текущем активном периоде.

Управление роутером через локальный RCI API Keenetic
(`http://localhost:79/rci`), авторизация для запросов с самого роутера не
требуется.

## Возможности

- Активный период работы по расписанию (`active_from` / `active_to`, с
  поддержкой интервалов через полночь).
- Настраиваемый интервал между проверками/попытками переподключения.
- Ограничение числа неудачных попыток подряд — после исчерпания программа
  ждёт следующего активного периода вместо бесконечных переподключений.
- Смещение местного времени (`utc_offset`), если системные часы роутера не в
  нужном часовом поясе (на Entware обычно UTC).
- Автоопределение WAN-интерфейса через `/rci/show/wans` либо явное указание в
  конфиге.
- Несколько сервисов проверки внешнего IP с перебором по порядку при отказе.
- Флаг `--test` — разовая диагностическая проверка без запуска демона.
- Статический бинарник без зависимостей, кросс-компиляция под
  mips / mipsel / aarch64.

## Конфигурация

Файл `white-ip.conf` ищется рядом с исполняемым файлом. Формат — простой
`key = value`, строки с `#` и пустые строки игнорируются. Пример — в
[`white-ip.conf.example`](white-ip.conf.example):

```ini
active_from = 08:00
active_to = 23:00

interval_seconds = 60

max_attempts = 10

wan_interface =

check_urls = https://api.ipify.org,https://ipv4.icanhazip.com,https://checkip.amazonaws.com

utc_offset = +3
```

| Параметр | Обязателен | Описание |
|---|---|---|
| `active_from`, `active_to` | да | Активный период в формате `ЧЧ:ММ`, местное время. Поддерживается диапазон через полночь (например `23:00` → `06:00`). |
| `interval_seconds` | да | Интервал между проверками и между попытками переподключения, в секундах. |
| `max_attempts` | да | Сколько неудачных попыток подряд допускается в рамках одного активного периода, прежде чем программа перестанет пытаться до следующего периода. |
| `wan_interface` | нет | ID WAN-интерфейса (например `GigabitEthernet1`). Если не задан — определяется автоматически через `/rci/show/wans`. |
| `check_urls` | нет | Список сервисов проверки внешнего IP через запятую, пробуются по очереди. По умолчанию: `api.ipify.org`, `ipv4.icanhazip.com`, `checkip.amazonaws.com`. |
| `utc_offset` | нет | Смещение местного времени от UTC в часах, например `+11` или `-1`. Если не задан — используется системное время как есть. |

Конфиг перечитывается на каждой итерации цикла, перезапуск демона для
применения изменений не требуется.

## Флаг --test

Разовая диагностика без запуска демона и без переключения интерфейса:

```
./white-ip --test
```

Выводит системное и местное (с учётом `utc_offset`) время, параметры таймера
из конфига, определённый WAN-интерфейс, его текущий IP, внешний IP и итоговый
вердикт — белый или серый адрес.

Пример вывода:

```
Системное время: 2026-07-06 10:30:50 UTC
Местное время: 2026-07-06 21:30:50 UTC+11
Параметры таймера из конфигурации:
  активный период: 08:00 - 23:00
  интервал между попытками: 1m0s
  макс. число попыток подряд: 10
  смещение местного времени: UTC+11
WAN-интерфейс: GigabitEthernet1
IP на WAN: 192.168.10.151
IP на внешнем ресурсе: 109.100.100.100
Результат: серый IP
```

## Сборка

Нужен установленный Go.

Сборка под все три архитектуры (mipsel, mips, aarch64 — актуальные платформы
Entware на Keenetic) через `make`:

```
make            # то же, что make all
make mipsel     # только под mipsel
make mips       # только под mips
make aarch64    # только под aarch64
make clean      # удалить bin/
```

Собрать только под текущую архитектуру хоста:

```
go build -o white-ip .
```

## Установка на роутер

1. Скопировать бинарник нужной архитектуры и `white-ip.conf` (на основе
   `white-ip.conf.example`) в одну директорию на роутере, например
   `/opt/root/`.
2. Сделать бинарник исполняемым: `chmod +x white-ip_mipsel-3.4`.
3. Проверить конфигурацию: `./white-ip_mipsel-3.4 --test`.
4. Запустить демон: `./white-ip_mipsel-3.4`.

## Автозапуск
Для автозапуска при загрузке роутера потребуется init.d-скрипт Entware:
1. Положить файл S99white-ip в `/opt/etc/init.d/S99white-ip`
2. Выдать права: `chmod +x /opt/etc/init.d/S99white-ip`
3. Узнать команды управления: `/opt/etc/init.d/S99white-ip`


## Как это работает изнутри

- `rci.go` — минимальный клиент RCI: определение WAN-интерфейса
  (`/rci/show/wans`), чтение состояния интерфейса
  (`/rci/show/interface/<id>`), включение/выключение интерфейса
  (`POST {"interface":{"<id>":{"up":bool}}}`).
- `externalip.go` — запрос внешнего IP с перебором сервисов при отказе.
- `config.go` — парсинг `white-ip.conf`, окно активности с учётом перехода
  через полночь, пересчёт местного времени по `utc_offset`.
- `main.go` — точка входа, флаг `--test`, основной цикл демона.
